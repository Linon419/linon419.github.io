如果一个事物包含另一个事物，那么这就是一个类内部包含另一个类

例如：人体和心脏，汽车和发动机

分类

  1. 成员内部类
  2. 局部内部类（包含匿名内部类）

      }

权限修饰符的小结：

public > protected > (default) > private

  1. 外部类：public，default
  2. 成员内部类：都行
  3. 局部内部类：都不行

匿名内部类

如果接口的实现了类（或父类的子类），只需要使用唯一一次

那么该情况下可省略该类的定义，称为匿名内部类

格式：

1
2
3

接口名称 对象名 = new 接口名称(){
  //覆盖重写抽象方法
};

public interface MyInterface{
  
  public void method(){
    System.out.println("这是接口");
  }
}


public class Demo{
  
  public static void main(String[] args){
    
    MyInterface obj = new MyInterface(){
      @override
      public void method(){
        
        System.out.println("这是匿名内部类");
      }
      
    };
    obj.method();//这是匿名内部类
  }
}

对格式"new 接口名称(){…}“解析

  1. new 代表创建对象动作
  2. 接口名称就是匿名内部类需要实现哪个接口
  3. {///…}是匿名内部类内容

另外几点

  1. 匿名内部类在【创建对象】时，只能使用唯一一次

如果希望多次创建对象。而且类内容一样，需要单独定义实现类

  1. 匿名对象在调用方法时，只能调用唯一一次

如果希望同一对象，调用多个方法，必须给对象起名字

  1. 匿名内部类省略了实现类，匿名对象省略了【对象名称】。两个不是一回事

    MyInterface obj = new MyInterface(){

        System.out.println("这是匿名内部类");

      }
      
    }.method();
  }
}
